# 隐藏附件入口设计

## 背景

当前用户端 `ComposerBar` 保留了附件按钮，但附件上传流程尚未接入。现阶段需要隐藏用户可见入口，同时保留现有按钮代码，便于后续恢复和接入上传能力。

## 目标

- 用户端不显示“添加附件”按钮，也不让其出现在可访问性树中。
- 保留附件按钮、图标和标签代码，不删除未来可复用的入口实现。
- 隐藏入口后，输入框和 loading 状态占用原附件按钮释放的空间，发送按钮保持在最右侧。
- 不改变消息发送 payload、检索模式或后端接口。

## 方案

在 `ComposerBar.vue` 中增加明确的前端可见性开关，默认关闭附件入口。模板通过该开关决定是否渲染附件按钮，因此隐藏状态不会留下无效控件或可访问性节点。CSS 使用“附件入口可见/隐藏”的布局契约：隐藏时采用两列布局，输入框和 loading 位于第 1 列，发送按钮位于第 2 列；开启时恢复三列布局。

附件按钮的 `Paperclip` 引入、`attachmentLabel` 和按钮模板保留，不实现上传行为，也不引入新的后端依赖。

## 测试

- 组件测试确认默认状态下附件按钮和“添加附件”标签不可见。
- 组件测试确认现有固定 `deep` 模式提交行为不变。
- 源码样式契约测试确认隐藏状态使用两列布局，搜索 loading 的叠层仍位于输入区域，发送按钮位置稳定。
- 运行前端全量测试、类型检查、生产构建和 `git diff --check`。

## 非目标

- 不删除附件相关代码。
- 不实现文件选择、上传、解析或后端接口。
- 不修改管理端、历史数据或检索架构。
